# SkyCore Pro - Backend Architecture Guide

## 📋 Overview

The SkyCore Pro backend is a Python-based service layer that provides:
- **REST API** for drone management
- **WebSocket** for real-time telemetry
- **MAVLink Protocol** for drone communication
- **Database** for persistence
- **Signal Processing** for RF analysis

## 🚀 Quick Start

### Option 1: Docker (Recommended)

```bash
# Clone repository
git clone https://github.com/your-org/skycore-pro.git
cd skycore-pro/backend

# Copy environment file
cp .env.example .env

# Edit .env with your configuration
nano .env

# Start all services
docker-compose up -d

# Check status
docker-compose ps
```

### Option 2: Manual Installation

```bash
# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Set environment variables
export DATABASE_URL="postgresql://user:pass@localhost/skycore"
export REDIS_URL="redis://localhost:6379"

# Run database migrations
alembic upgrade head

# Start server
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## 📡 API Endpoints

### Drones

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/drones` | List all drones |
| POST | `/api/v1/drones` | Add new drone |
| GET | `/api/v1/drones/{id}` | Get drone details |
| PUT | `/api/v1/drones/{id}` | Update drone |
| DELETE | `/api/v1/drones/{id}` | Remove drone |
| POST | `/api/v1/drones/{id}/connect` | Connect to drone |
| POST | `/api/v1/drones/{id}/command` | Send command |

### Missions

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/missions` | List missions |
| POST | `/api/v1/missions` | Create mission |
| GET | `/api/v1/missions/{id}` | Get mission |
| PUT | `/api/v1/missions/{id}` | Update mission |
| POST | `/api/v1/missions/{id}/execute` | Start mission |
| POST | `/api/v1/missions/{id}/abort` | Abort mission |

### Telemetry

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/telemetry/{drone_id}` | Get current telemetry |
| GET | `/api/v1/telemetry/{drone_id}/history` | Get telemetry history |
| GET | `/api/v1/telemetry/stream` | Stream telemetry (WS) |

### C-UAS

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/threats` | List detected threats |
| GET | `/api/v1/threats/{id}` | Get threat details |
| POST | `/api/v1/threats/{id}/respond` | Respond to threat |
| GET | `/api/v1/frequency-scan` | Start frequency scan |
| GET | `/api/v1/airspace` | Get airspace status |

### Settings

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/settings` | Get all settings |
| PUT | `/api/v1/settings` | Update settings |
| GET | `/api/v1/settings/regulatory` | Get regulatory config |
| PUT | `/api/v1/settings/regulatory` | Update regulatory config |

### Connection
```javascript
ws://server:8000/ws/telemetry?token=xxx
```

### Message Format
```json
{
  "type": "telemetry",
  "timestamp": 1704067200000,
  "drone_id": "drone-1",
  "data": {
    "latitude": 32.0853,
    "longitude": 34.7818,
    "altitude": 120,
    "speed": 15.5,
    "heading": 45,
    "battery": 78,
    "signal": 92
  }
}
```

### Subscriptions
- `telemetry/{drone_id}` - Drone telemetry
- `threats` - Threat detection updates
- `aircraft` - ADS-B aircraft positions
- `system` - System status

## 🗄️ Database Schema

### Tables

```sql
-- Drones
CREATE TABLE drones (
    id UUID PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    model VARCHAR(100),
    manufacturer VARCHAR(100),
    status VARCHAR(50),
    last_position GEOMETRY(Point, 4326),
    last_seen TIMESTAMP,
    config JSONB,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Missions
CREATE TABLE missions (
    id UUID PRIMARY KEY,
    name VARCHAR(200),
    drone_id UUID REFERENCES drones(id),
    status VARCHAR(50),
    waypoints JSONB,
    created_at TIMESTAMP DEFAULT NOW(),
    started_at TIMESTAMP,
    completed_at TIMESTAMP
);

-- Telemetry History
CREATE TABLE telemetry (
    id BIGSERIAL PRIMARY KEY,
    drone_id UUID REFERENCES drones(id),
    timestamp TIMESTAMP NOT NULL,
    position GEOMETRY(Point, 4326),
    altitude DECIMAL,
    speed DECIMAL,
    heading DECIMAL,
    battery DECIMAL,
    signal DECIMAL,
    raw_data JSONB
);
CREATE INDEX idx_telemetry_drone_time ON telemetry(drone_id, timestamp DESC);

-- Threats
CREATE TABLE threats (
    id UUID PRIMARY KEY,
    type VARCHAR(50),
    position GEOMETRY(Point, 4326),
    threat_level INTEGER,
    confidence DECIMAL,
    protocol VARCHAR(50),
    rf_signature VARCHAR(200),
    status VARCHAR(50),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Flight Logs
CREATE TABLE flight_logs (
    id UUID PRIMARY KEY,
    drone_id UUID REFERENCES drones(id),
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    max_altitude DECIMAL,
    max_distance DECIMAL,
    battery_start DECIMAL,
    battery_end DECIMAL,
    waypoints JSONB,
    incidents JSONB
);
```

## 🔧 Hardware Configuration

### RTL-SDR (ADS-B/Scanner)
```python
# config/hardware.yaml
sdr:
  adsb:
    device_index: 0
    sample_rate: 2000000
    frequency: 1090000000
    gain: 49.6
  scanner:
    device_index: 0
    sample_rate: 2000000
    frequency_ranges:
      - [2400000000, 2500000000]  # 2.4 GHz
      - [5725000000, 5875000000]  # 5.8 GHz
```

### DJI OcuSync
```python
# config/hardware.yaml
dji:
  sdk_path: "/dev/ttyUSB0"
  baudrate: 115200
  protocol: "OcuSync 4.0"
```

### u-blox GNSS/RTK
```python
# config/hardware.yaml
gnss:
  port: "/dev/ttyACM0"
  baudrate: 460800
  constellations:
    - GPS
    - GLONASS
    - Galileo
    - BeiDou
  rtcm:
    enabled: true
    server: "rtk2go.com"
    port: 2101
```

## 🛡️ Security

### Authentication
```python
# JWT Token structure
{
  "sub": "user_id",
  "role": "admin|operator|viewer",
  "exp": 1704153600,
  "permissions": ["drones:read", "missions:write"]
}
```

### API Keys
- Admin API Key: Full access
- Operator API Key: Drones + Missions
- Viewer API Key: Read-only

## 🔄 CI/CD Pipeline

```yaml

name: Deploy

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run tests
        run: pytest tests/
      - name: Build Docker
        run: docker build -t skycore-pro:latest .

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to server
```
