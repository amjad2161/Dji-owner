## Coordinate Systems

### NED Frame (North-East-Down)

The navigation frame uses NED (North-East-Down) convention:

```
Position vector: P = [North, East, Down]ᵀ in meters
Velocity vector: V = [vN, vE, vD]ᵀ in m/s
Acceleration:   A = [aN, aE, aD]ᵀ in m/s²
```

### Body Frame

The body frame is aligned with the drone:

```
x-axis: Forward (roll axis)
y-axis: Right (pitch axis)
z-axis: Down (yaw axis)
```

### Rotation Convention

Rotation from navigation to body frame:

```
R_nb = R(φ, θ, ψ)
```

Where φ = roll, θ = pitch, ψ = yaw.

## Quaternion Mathematics

### Quaternion Representation

A quaternion q is represented as:

```
q = [w, x, y, z]ᵀ = w + xi + yj + zk
```

With constraint: w² + x² + y² + z² = 1

### Quaternion Multiplication

For quaternions q₁ = [w₁, x₁, y₁, z₁]ᵀ and q₂ = [w₂, x₂, y₂, z₂]ᵀ:

```
q₁ ⊗ q₂ = [
    w₁w₂ - x₁x₂ - y₁y₂ - z₁z₂,
    w₁x₂ + x₁w₂ + y₁z₂ - z₁y₂,
    w₁y₂ - x₁z₂ + y₁w₂ + z₁x₂,
    w₁z₂ + x₁y₂ - y₁x₂ + z₁w₂
]ᵀ
```

### Quaternion to Rotation Matrix

Given quaternion q = [w, x, y, z]ᵀ:

```
R = [
    1 - 2(y² + z²),    2(xy - wz),        2(xz + wy),
    2(xy + wz),         1 - 2(x² + z²),    2(yz - wx),
    2(xz - wy),         2(yz + wx),        1 - 2(x² + y²)
]
```

### Rotation Matrix to Quaternion

Using the trace of R:

```
tr(R) = r₁₁ + r₂₂ + r₃₃ = 4w² - 1

if w ≥ 0:
    w = 0.5√(tr(R) + 1)
    x = (r₃₂ - r₂₃) / (4w)
    y = (r₁₃ - r₃₁) / (4w)
    z = (r₂₁ - r₁₂) / (4w)
else:
    x = 0.5√(1 + r₁₁ - r₂₂ - r₃₃)
    y = 0.5√(1 - r₁₁ + r₂₂ - r₃₃)
    z = 0.5√(1 - r₁₁ - r₂₂ + r₃₃)
```

### Quaternion Integration (First Order)

```
q(t + dt) = q(t) + Ω(q)q(t) · dt/2

where Ω(q) = [
    0,    -ωx,  -ωy,  -ωz
    ωx,   0,    ωz,   -ωy
    ωy,   -ωz,  0,    ωx
    ωz,   ωy,   -ωx,  0
]
```

## 22-State Augmented Unscented Kalman Filter (AUKF)

### State Vector (22 dimensions)

```
x = [pN, pE, pD, vN, vE, vD, qw, qx, qy, qz,
     bgx, bgy, bgz, bax, bay, baz, wN, wE, wD,
     cb, cd, misc₁, misc₂]ᵀ

Position:      p = [pN, pE, pD]ᵀ      (3)
Velocity:      v = [vN, vE, vD]ᵀ      (3)
Quaternion:    q = [qw, qx, qy, qz]ᵀ  (4)
Gyro bias:    bg = [bgx, bgy, bgz]ᵀ  (3)
Accel bias:    ba = [bax, bay, baz]ᵀ  (3)
Wind:          w = [wN, wE, wD]ᵀ      (3)
Clock bias:    cb                    (1)
Clock drift:   cd                    (1)
Miscellaneous: misc₁, misc₂          (2)
                              ──────────
                              Total:    22
```

### UKF Parameters (α=0.001, κ=0, β=2)

```
α = 0.001    (spread parameter)
κ = 0        (secondary parameter)
β = 2        (prior knowledge, 2 for Gaussian)

λ = α²(n + κ) - n
  = (0.001)²(22 + 0) - 22
  = -21.999978

n = 22 (state dimension)

Sigma point weights:
γ = √(n + λ) = √22.000001 ≈ 4.690
```

### Sigma Points Generation

For state dimension n = 22, generate 2n + 1 = 45 sigma points:

```
χ⁰ = x̄                                    (weight w⁰_m = λ/(n+λ))
χⁱ = x̄ + γ·Pₖ¹/² · eᵢ                    (weight wⁱ_m = 1/(2(n+λ)))
χⁱ⁺ⁿ = x̄ - γ·Pₖ¹/² · eᵢ                  (weight wⁱ⁺ⁿ_m = 1/(2(n+λ)))

where eᵢ is the i-th basis vector
      Pₖ¹/² is the matrix square root of covariance
```

### UKF Algorithm

#### Prediction Step

1. **Sigma Point Transform**:
   ```
   χₖ⁽ⁱ⁾ = f(χₖ₋₁⁽ⁱ⁾, uₖ₋₁)

   χₖ⁽ⁱ⁾ = [
       pₖ₋₁ + vₖ₋₁·dt
       vₖ₋₁ + R(qₖ₋₁)·(a - ba)·dt
       qₖ₋₁ ⊗ q(ω - bg)·dt
       bg
       ba
       w
       cb
       cd + δcd·dt
   ]
   ```

2. **Mean Prediction**:
   ```
   x̄ₖ = Σᵢ wⁱ_m · χₖ⁽ⁱ⁾
   ```

3. **Covariance Prediction**:
   ```
   Pₖ = Σᵢ wⁱ_c · (χₖ⁽ⁱ⁾ - x̄ₖ)(χₖ⁽ⁱ⁾ - x̄ₖ)ᵀ + Qₖ

   Process noise Qₖ = diag([
       σ²_p, σ²_p, σ²_p,      # Position noise
       σ²_v, σ²_v, σ²_v,      # Velocity noise
       σ²_q, σ²_q, σ²_q, σ²_q,  # Quaternion noise
       σ²_bg, σ²_bg, σ²_bg,   # Gyro bias noise
       σ²_ba, σ²_ba, σ²_ba,   # Accel bias noise
       σ²_w, σ²_w, σ²_w,      # Wind noise
       σ²_cb, σ²_cd,          # Clock noise
       σ²_m, σ²_m             # Misc noise
   ])
   ```

#### Update Step

1. **Measurement Sigma Points**:
   ```
   Ŷₖ⁽ⁱ⁾ = h(χₖ⁽ⁱ⁾)
   ```

2. **Predicted Measurement**:
   ```
   ŷ̄ₖ = Σᵢ wⁱ_m · Ŷₖ⁽ⁱ⁾
   ```

3. **Innovation Covariance**:
   ```
   Sₖ = Σᵢ wⁱ_c · (Ŷₖ⁽ⁱ⁾ - ŷ̄ₖ)(Ŷₖ⁽ⁱ⁾ - ŷ̄ₖ)ᵀ + Rₖ
   ```

4. **Cross-Covariance**:
   ```
   Cₖ = Σᵢ wⁱ_c · (χₖ⁽ⁱ⁾ - x̄ₖ)(Ŷₖ⁽ⁱ⁾ - ŷ̄ₖ)ᵀ
   ```

5. **Kalman Gain**:
   ```
   Kₖ = Cₖ · Sₖ⁻¹
   ```

6. **State Update**:
   ```
   x̄ₖ⁺ = x̄ₖ + Kₖ(yₖ - ŷ̄ₖ)
   Pₖ⁺ = Pₖ - KₖSₖKₖᵀ
   ```

### Adaptive Noise Estimation

The AUKF includes adaptive noise estimation:

```
Adaptive process noise scaling:
Q̄ₖ = α_adapt · Qₖ

where α_adapt = max(1, χ²/(n·ν))
      χ² = Σᵢ (yₖ - ŷ̄ₖ)ᵀ Sₖ⁻¹ (yₖ - ŷ̄ₖ)
      ν = measurement dimension
```

### Square-Root Implementation

For numerical stability, use square-root formulation:

```
Pₖ = SₖSₖᵀ  (Cholesky decomposition)

Sₖ⁺ = cholupdate(Sₖ, Kₖ, -1)
```

## SE(3) Geometric Control

### Error Definition on SE(3)

The tracking error is defined using rotation matrices and positions.

**Rotation Error**:
```
Rₑ = Rₐᵀ Rₐᵈ

where Rₐ = current attitude rotation matrix
      Rₐᵈ = desired attitude rotation matrix
```

**Position Error**:
```
e_p = p - p_d

where p = current position in NED
      p_d = desired position in NED
```

### Attitude Error on SO(3)

The attitude error can be expressed using quaternion:

```
e_R = 0.5(Vec(Rₑ - Rₑᵀ))
    = [e_Rx, e_Ry, e_Rz]ᵀ

Using quaternion representation:
e_R = (q_d⁻¹ ⊗ q)Vec
```

### SE(3) Geometric Controller Law

The SE(3) geometric controller computes:

```
τ = -k_R · e_R - k_ω · e_ω + ω × Jω + J · ω̇_d

where:
τ       = control torque vector [τx, τy, τz]ᵀ
k_R     = rotation gain (positive scalar)
k_ω     = angular rate gain (positive scalar)
e_R     = rotation error vector (3×1)
e_ω     = angular rate error vector (3×1)
ω       = current angular velocity [ωx, ωy, ωz]ᵀ
J       = inertia matrix (3×3 symmetric)
ω × Jω  = centrifugal coupling term
ω̇_d     = desired angular acceleration
```

### Cascaded Position-Velocity-Attitude Control

The full cascaded control architecture:

#### Level 1: Position Control
```
F_des = -k_p · e_p - k_v · e_v + m·g·e_d + m·ω̇_d

where:
F_des = desired thrust vector (NED frame)
k_p   = position gain
k_v   = velocity gain
m     = drone mass (kg)
g     = gravity (9.81 m/s²)
e_d   = down unit vector [0, 0, 1]ᵀ
```

#### Level 2: Attitude Control
```
τ = Rₑᵀ · τ_body

where τ_body computed via SE(3) law above
```

#### Level 3: Motor Mixing

```
u = T · τ

where u = [u₁, u₂, u₃, u₄]ᵀ motor outputs
      T = motor mixing matrix for X-configuration:

      T = [
          1,   1,   1,   1,    # Total thrust
          0,   d,   0,  -d,    # Roll moment
         -d,   0,   d,   0,    # Pitch moment
         -c,   c,  -c,   c     # Yaw moment
      ]

      d = arm length (m)
      c = yaw control factor
```

### Minimum Snap Trajectory Generation

For smooth trajectory between waypoints:

**Quintic Polynomial**:
```
p(t) = a₀ + a₁t + a₂t² + a₃t³ + a₄t⁴ + a₅t⁵

Boundary conditions:
p(t₀) = p₀, ṗ(t₀) = v₀, p̈(t₀) = a₀
p(t₁) = p₁, ṗ(t₁) = v₁, p̈(t₁) = a₁
```

**Snap Minimization** (4th derivative):

```
Minimize: J = ∫(p⁽⁴⁾(t))² dt from t₀ to t₁

Solution: Solve 6×6 system for coefficients [a₀,...,a₅]
```

## Path Planning Algorithms

### A* Path Planning

**Algorithm**:
```
f(n) = g(n) + h(n)

where:
f(n) = total estimated cost through node n
g(n) = cost from start to node n
h(n) = heuristic estimated cost from n to goal
```

**Heuristics**:
```
Euclidean: h(n) = √((n.x-g.x)² + (n.y-g.y)² + (n.z-g.z)²)

Manhattan: h(n) = |n.x-g.x| + |n.y-g.y| + |n.z-g.z|

Octile:     h(n) = min(dx,dy,dz)·(√2-1) + dx+dy+dz - min(dx,dy,dz)
```

**3D Connectivity**: 26-connected neighbors

### RRT* Path Planning

**Algorithm**:
```
1. Sample random configuration x_rand
2. Find nearest node x_nearest in tree
3. Steer from x_nearest towards x_rand with step_size
4. If collision-free, add new node x_new
5. Find nodes within rewire_radius of x_new
6. Choose parent with minimum cost
7. Rewire tree if paths improve
8. Repeat until goal reached or max_iterations
```

**Rewiring Radius**:
```
γ = γ_opt · (log(n)/n)^(1/d)

where:
γ_opt ≈ 2 · (π)^(1/2) for 2D
      ≈ 2 · (3π)^(1/3) for 3D
n = number of nodes
d = state dimension
```

**Informed RRT***:
```
Ellipsoid sampling:

x_sample = center + R · (L · u)

where:
center = (x_start + x_goal) / 2
R      = rotation matrix (start→goal direction)
L      = diagonal matrix [c, c, h/2]
c      = c_max / 2 (half length)
h      = 2·√(c² - d²) (height)
d      = ||x_start - x_goal||
u      = random point in unit sphere
```

### Velocity Obstacle (VO)

**Velocity Obstacle Definition**:
```
VO_A,B = {v | λ(A,B,v) < τ}

where:
λ(A,B,v) = time to collision with obstacle B
         = -((p_B - p_A)·(v_A - v_B)) / ||v_A - v_B||²

τ = time horizon
```

**VO Cone Construction**:
```
1. Relative position: p_rel = p_A - p_B
2. Combined radius: R = r_A + r_B + margin
3. Cone half-angle: θ = arcsin(R / ||p_rel||)
4. Cone edges: direction ± θ from p_rel
```

**Safe Velocity Selection**:
```
v_safe = argmin ||v - v_pref||

subject to:
||v|| ≤ v_max
||v - v_current|| ≤ a_max · τ_collision
v ∉ VO
```

### ORCA (Optimal Reciprocal Collision Avoidance)

**ORCA Half-Plane**:
```
For agent A and B:
n · v_A ≥ b

where:
n = (p_B - p_A) / ||p_B - p_A|| (outward normal)
b = (v_B - v_A + (R/τ)·n)·n / 2   (offset)

τ = time horizon
R = combined radius
```

**Linear Program**:
```
minimize: ||v - v_preferred||²

subject to:
n_i · v ≥ b_i  ∀ constraints
||v|| ≤ v_max
```

## Visual-Inertial Odometry (VIO)

### MSCKF State Augmentation

**IMU Preintegration**:
```
Δp = Σ (Δv·dt + ½R·(a - b_a)·dt²)
Δv = Σ R·(a - b_a)·dt
Δq = ⊗ q(ω - b_g)·dt
```

**Feature Triangulation**:
```
Triangulate from multiple views:
x_feat = argmin Σ ||P_i · x_feat - p_i||²

where P_i = projection matrix for view i
```

**MSCKF Update**:

For M camera observations of a 3D feature:

```
Construct measurement:
z = [u₁, v₁, ..., u_M, v_M]ᵀ

Jacobian linearization:
z = Hx · x + Hf · f + n

Eliminate feature f:
H_o · x = z - n

Standard EKF update with H_o
```

### VIO Initialization

**Accelerometer Calibration**:
```
g = mean(accel) - mean(gyro × ω)

Initialize biases:
b_g = mean(gyro) initial
b_a = mean(accel) - [0, 0, g]ᵀ
```

## Geodesy and Navigation

### Haversine Distance

```
a = sin²(Δlat/2) + cos(lat₁)·cos(lat₂)·sin²(Δlon/2)
c = 2·atan2(√a, √(1-a))

d = R_Earth · c

where:
R_Earth ≈ 6371000 m
Δlat = lat₂ - lat₁
Δlon = lon₂ - lon₁
```

### LLA to ECEF Conversion

```
X = (N + h) · cos(lat) · cos(lon)
Y = (N + h) · cos(lat) · sin(lon)
Z = (N·(1-e²) + h) · sin(lat)

where:
N = R_E / √(1 - e²·sin²(lat))  (radius of curvature)
e² = 2f - f²  (first eccentricity squared)
f = 1/298.257223563  (WGS84 flattening)
```

### ECEF to NED Conversion

```
NED = R_ne · (ECEF - ECEF_origin)

where R_ne = [
    -sin(lat)·cos(lon),  -sin(lat)·sin(lon),  cos(lat)
    -sin(lon),            cos(lon),           0
    -cos(lat)·cos(lon),  -cos(lat)·sin(lon), -sin(lat)
]
```

## YOLOv8 Detection

### Bounding Box IoU

```
IoU(A, B) = |A ∩ B| / |A ∪ B|

where:
Intersection: x1 = max(A.x1, B.x1), y1 = max(A.y1, B.y1)
              x2 = min(A.x2, B.x2), y2 = min(A.y2, B.y2)

if x2 ≤ x1 or y2 ≤ y1: IoU = 0
```

### CIoU Loss (Complete IoU)

```
CIoU = IoU - (ρ²(b,b')/c²) - αv

where:
ρ²(b,b') = center distance squared
c        = diagonal of enclosing box
α        = trade-off parameter
v        = aspect ratio similarity
```
